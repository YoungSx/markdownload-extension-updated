    Welcome to MarkSnip!

    Thanks for installing MarkSnip. This guide will help you get started with clipping web pages as clean Markdown.

    Start with the Quick Start section below, or browse the table of contents on the left.

    #Quick Start

    1. Click the MarkSnip icon in the browser toolbar to open the popup.
    2. The current page is run through Mozilla Readability to extract the main content, then converted to Markdown with Turndown.
    3. Edit, copy, or download the Markdown using the built-in CodeMirror editor.
    Tip: Use Alt+Shift+M to open the popup without clicking the icon.

    #Using the Popup

    #Quick Settings

    At the top of the popup you'll find two toggle switches:

    • Images — when on, images are downloaded alongside your Markdown file (requires Downloads API mode).
    • Template — when on, the front-matter and back-matter templates are prepended/appended to the output.

    #Clipping Selection vs. Full Document

    Below the toggles is a segmented control with two buttons:

    • Selection — clips only the text you selected on the page before opening the popup.
    • Document — clips the entire page content (default if nothing is selected).

    Use Pick Element when the automatic article extraction misses part of a page. MarkSnip will switch to the page, let you choose an element with an inspect-style picker, and then load that element as Markdown in the popup for review or copy it directly, depending on your Element Picker setting.

    #Editing the Title

    The Title field determines the filename of the downloaded Markdown file. It is populated automatically from the title template (configurable in options) but you can edit it freely before saving.

    #Markdown Preview

    The main area of the popup is a syntax-highlighted Markdown editor (powered by CodeMirror). You can make quick edits here before downloading.

    Above the editor are two buttons:

    • Copy All — copies all the Markdown to your clipboard.
    • Copy Selection — appears when you select text inside the editor, allowing you to copy just the highlighted portion.

    #Action Buttons

    ButtonAction
    Main ActionUses your configured popup default action. Choose Markdown (.md), Plain text (.txt), HTML (.html), PDF, or Send to Assistant.
    Selection ActionAppears when text is selected inside the editor; exports or sends only the selected text using the same popup default action.
    Send to ObsidianCopies Markdown to clipboard and opens Obsidian via the Advanced URI plugin to create a new note (only visible when Obsidian integration is enabled).

    Note: This popup default action setting only affects popup actions. Context menus, keyboard download shortcuts, batch exports, Library exports, Agent Bridge, and Obsidian actions remain Markdown-based.

    #Batch Processing

    Click the batch icon in the popup header to switch to batch mode.

    #Adding URLs

    Paste URLs into the text area, one per line. You can also use Markdown link syntax:

    https://example.com/page1
https://example.com/page2
[Page Title](https://example.com/page3)

    #Pick Links from Page

    Click Pick Links from Page to enter the visual link picker. This injects an overlay onto the current page that lets you click on links to select them. A floating toolbar shows how many links you've selected, and a Done button sends them back to the batch URL list.

    #Output Format

    • ZIP file — all converted Markdown files are bundled into a single .zip download.
    • Individual — each page is downloaded as a separate .md file.

    Click Convert All URLs to start processing. A progress bar shows the current status including a count and the URL being processed.

    #Context Menus & Shortcuts

    Right-click on a page to access MarkSnip actions under the context menu. These are available when Enable Context Menus is turned on in settings.

    #Settings & Customization

    Open the options page by clicking the gear icon in the popup header, or by right-clicking the MarkSnip icon and selecting Options.

    #Templates

    Title Template

    Controls the popup's title field and the resulting filename. Uses Custom Text Substitutions. Default: {pageTitle}

    Downloads Subfolder

    A subfolder within your browser's downloads folder where Markdown files are saved. Only available in Downloads API mode. Uses text substitutions.

    Disallowed Characters

    Characters automatically stripped from filenames, in addition to system-reserved characters. Default: []#^ (for Obsidian compatibility).

    Front/Back Templates

    Text prepended or appended to every clipped Markdown file. Useful for YAML front-matter or other metadata. Uses text substitutions.

    ---
created: {date:YYYY-MM-DDTHH:mm:ss} (UTC {date:Z})
tags: [{keywords}]
source: {pageURL}
author: {byline}
---

    #Downloads

    Context Menus

    Toggles the right-click context menu items on and off. Use Visible Menu Items to choose the individual MarkSnip actions that are allowed to appear. Browsers only show items that apply to where you right-click, such as page, selection, link, image, or tab menus. Default: On, with all menu items visible.

    Default Popup Action

    Controls the popup's primary action buttons. Choose Markdown, Plain text, HTML, PDF, or Send to Assistant. This does not change context-menu downloads, keyboard download shortcuts, batch output, Library exports, Agent Bridge output, or Obsidian sends.

    Default Assistant Target

    When Send to Assistant is selected, choose the popup's default destination such as ChatGPT, Claude, Perplexity, or one of your custom assistant URLs.

    Assistant Targets

    ChatGPT, Claude, and Perplexity are built in. Add custom HTTPS assistant URLs that accept a {prompt} placeholder in the query string. MarkSnip inserts the current Markdown into that placeholder when you send from the popup.

    Assistant URL Length Cap

    Controls how long a generated assistant URL can be before MarkSnip falls back to copying the Markdown to your clipboard and opening the assistant landing page instead. The default is 3600, and changing it is not recommended.

    #Markdown Options

    Heading Style

    Setext uses underlines (=== / ---); Atx (default) uses # prefixes.

    Horizontal Rule Style

    Choose between ***, ---, or ___ (default).

    Bullet List Marker

    *, - (default), or +.

    Code Block Style

    Indented (four-space indent) or Fenced (default, wrapped in ``` or ~~~).

    Preserve Code Block HTML Formatting

    When enabled, preserves the original HTML formatting inside code blocks. Default: Off.

    Emphasis & Strong Delimiters

    Italics: _ (default), *, or __ (for Roam). Bold: ** (default) or __.

    Link Style

    Inlined (default): [Google](http://google.com). Referenced: separate link definitions at the bottom. Strip Links: removes links entirely.

    Image Style

    StyleOutputRequires Download Images
    Original Source![](http://example.com/img.jpg)No
    Strip Images(image removed)No
    Pure Markdown (default)![](folder/image.jpg)Yes
    Base64 Encoded![](data:image/png;base64,...)Yes
    Obsidian Internal Embed![[folder/image.jpg]]Yes
    Obsidian Embed (no folder)![[image.jpg]]Yes

    Escape Markdown Characters

    Backslash-escapes special Markdown characters to prevent misinterpretation. Default: On.

    Hashtag Handling

    Keep (default), Remove #, or Escape #. Only affects article text; Markdown syntax and code are preserved.

    #Obsidian & Downloads

    #Obsidian Integration

    Enable this to show the Send to Obsidian button and Obsidian context menu actions. Requires the Advanced Obsidian URI community plugin.

    • Vault Name — leave blank for the default vault.
    • Folder Name — supports text substitutions (e.g. {date:YYYY-MM-DD}/).
    Note: Images sent via Obsidian stay as remote links. The extension does not place attachment files inside the vault.

    #Download Mode

    ModeDescription
    Downloads API (recommended)Uses the browser's download API. Supports subfolders, image downloads, and Save As dialogs.
    Content LinkFalls back to a content link for downloading. More limited — disables image downloads and subfolders.

    Show Save As Dialog

    Forces the browser's Save As dialog for every download. Not recommended when Download Images is on.

    #Image Downloads

    When enabled (Downloads API mode only), images are downloaded alongside the Markdown file with local references.

    Image Filename Prefix

    A prefix or subfolder for downloaded images. Uses text substitutions. Including a / creates a subfolder. Default: {pageTitle}/.

    #Permissions & Privacy

    MarkSnip processes pages entirely within your browser. It does not send page content to any external server. This section explains what browser permissions MarkSnip uses, why they are needed, and when they are actually exercised.

    #Site Access

    MarkSnip requests access to web pages so it can clip pages across the web when you trigger a clip. This covers popup clipping, keyboard shortcuts, context-menu actions, multi-tab link actions, and batch processing of URLs opened in new tabs.

    • Only active when you clip — MarkSnip does not monitor browsing activity or read pages in the background.
    • Processed locally — page content is converted to Markdown inside your browser and is never sent to a remote service.
    • Managed by the browser — you can review this permission in your browser’s extension settings at any time.

    #Other Core Permissions

    Beyond site access, MarkSnip uses a small set of standard browser permissions to make its main features work. These permissions support local browser actions; they do not change the fact that clipping stays local.

    PermissionWhy MarkSnip uses itWhen it matters
    activeTabLets MarkSnip work with the page you explicitly invoked it on.Opening the popup or using a shortcut on the current tab.
    downloadsSaves Markdown files, ZIP exports, and downloaded images through the browser.Downloading clips or batch exports.
    clipboardWriteWrites generated Markdown to your clipboard.Using Copy actions.
    storageRemembers your settings, templates, themes, and feature toggles.Whenever you save or restore settings.
    contextMenusAdds right-click actions for pages, selections, links, images, and tabs.When context menus are enabled in settings.
    scriptingInjects MarkSnip’s capture helpers into the page at clip time.Whenever MarkSnip reads page content.
    offscreenUses a hidden document for Manifest V3 clipboard and download workflows in Chromium browsers.Behind the scenes during some copy and download actions.

    #Agent Bridge Permission

    The Agent Bridge feature requires an additional browser permission called Native Messaging. This is an optional permission — MarkSnip works perfectly without it.

    • What it does — lets MarkSnip talk to a small local companion program installed on your machine.
    • Local only — communication stays on your computer. No data leaves your machine.
    • When it’s used — only while Agent Bridge is enabled. Turning the toggle off stops all native messaging activity.
    • How to grant — enable the Agent Bridge toggle in Settings. MarkSnip will show an explanation before the browser’s permission prompt appears.
    • How to revoke — once granted, this permission cannot be selectively removed. To fully revoke it, you must reinstall the extension. Turning off the Agent Bridge toggle stops all native messaging activity without needing to revoke the permission.
    Note: If you deny the permission, regular clipping is completely unaffected. You can always try again later.

    #Troubleshooting / FAQ

    The popup shows "Processing page..." and nothing happens

    Some pages block content scripts. Try refreshing the page, or check that MarkSnip has permission to access the site. Internal browser pages (chrome://, about:) cannot be clipped.

    Images are not downloading

    Make sure Download Mode is set to "Downloads API" and the Images toggle is enabled. Content Link mode does not support image downloads.

    Send to Obsidian is not visible

    Enable Obsidian Integration in Settings. You also need the Advanced Obsidian URI community plugin installed in Obsidian.

    Context menu items are missing

    Check that Enable Context Menus is turned on in Downloads settings.

    Keyboard shortcuts don't work

    Another extension may have claimed the same shortcut. Open your browser's extension shortcut settings to reassign keys.

    Markdown output looks wrong

    Check the Markdown Options to adjust heading style, link style, code block style, and escape settings to match your workflow.

    #Custom Text Substitutions

    The Title Template, Downloads Subfolder, Front/Back Templates, Image Filename Prefix, and Obsidian Folder Name all support these variables:

    Article Metadata

    VariableDescription
    {title}Article title (as determined by Readability)
    {pageTitle}Title of the actual page (<title> tag)
    {length}Length of the article in characters
    {excerpt}Article description or short excerpt
    {byline}Author metadata
    {dir}Content direction (e.g. ltr)
    {baseURI}Parsed document/base URI
    {pageURL} / {tabURL}Address-bar URL of the tab
    {keywords}Meta keywords, comma-separated

    URL Components

    VariableDescription
    {origin}Scheme + domain + port
    {host}Hostname + port
    {hostname}Domain only
    {port}Port number
    {protocol}Protocol with trailing :
    {pathname}URL path
    {search}Query string including ?
    {hash}Fragment identifier including #

    All URL components are also available prefixed with page (e.g. {pageOrigin}, {pageHostname}).

    Date/Time

    Use {date:FORMAT} for the current date/time with Moment.js formatting tokens. Use {publishedTime:FORMAT} for article publication metadata when the page provides it.

    Meta Tags

    Any <meta> tag value can be referenced, e.g. {og:image}, {og:description}.

    Parameterizations

    Change the casing of any text variable (except {date} and {keywords}) by appending a colon and style:

    SyntaxStyleExample
    {variable:pascal}PascalCaseDifferentTypesOfCasing
    {variable:camel}camelCasedifferentTypesOfCasing
    {variable:kebab}kebab-casedifferent-types-of-casing
    {variable:snake}snake_casedifferent_types_of_casing
    {variable:lowercase}lowercaseall lowercase
    {variable:uppercase}UPPERCASEALL UPPERCASE
    {variable:ascii}ASCII onlyRemoves accents and non-ASCII characters
    {variable:slugify}URL slugLowercase, ASCII, alphanumeric or hyphen

    Filters chain left-to-right: {pageTitle:kebab:ascii} produces ASCII-only output (ASCII punctuation kept); use {pageTitle:slugify} for a fully URL-safe slug.

    Note: Not all websites provide all metadata values. Missing values are replaced with empty strings.
